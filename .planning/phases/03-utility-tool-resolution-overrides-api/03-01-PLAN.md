  <action>
Create lib/utilities.nix following the exact same module signature as runtimes.nix: `{ lib, pkgs }:` returning an attrset.

Unlike runtimes (which have version-specific attrs), utilities are single-version in nixpkgs. Every resolver function takes a `_version` argument (ignoring it) and returns the pkgs attr directly. This matches the pattern used for single-version runtimes like `rust = _version: pkgs.rustup;` in runtimes.nix.

The attrset MUST contain these exact keys and mappings:

```nix
{ lib, pkgs }:

{
  # CLI utilities -- mise tool name -> nixpkgs attr
  ripgrep   = _version: pkgs.ripgrep;
  rg        = _version: pkgs.ripgrep;      # alias
  fd        = _version: pkgs.fd;
  bat       = _version: pkgs.bat;
  jq        = _version: pkgs.jq;
  fzf       = _version: pkgs.fzf;
  git       = _version: pkgs.git;
  curl      = _version: pkgs.curl;
  wget      = _version: pkgs.wget;
  make      = _version: pkgs.gnumake;       # NOTE: pkgs.gnumake, not pkgs.make
  cmake     = _version: pkgs.cmake;
  gh        = _version: pkgs.gh;
  delta     = _version: pkgs.delta;
  eza       = _version: pkgs.eza;
  zoxide    = _version: pkgs.zoxide;
  starship  = _version: pkgs.starship;
  just      = _version: pkgs.just;
  hyperfine = _version: pkgs.hyperfine;
  tokei     = _version: pkgs.tokei;
}
```

Key points:
- `make` maps to `pkgs.gnumake` (not pkgs.make which does not exist in nixpkgs)
- `rg` is an alias for `ripgrep` (both should work since mise uses the tool name from the backend)
- All resolvers take `_version` (underscore prefix = unused) and return the package directly
- No version checking needed -- utilities use whatever version nixpkgs provides at the pin
