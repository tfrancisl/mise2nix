Rewrite lib/default.nix to implement the complete resolution pipeline. The new function signature adds optional extraPackages and overrides arguments:

```nix
{ lib }:
{
  fromMiseToml = path: { pkgs, extraPackages ? [], overrides ? {} }:
    let
      runtimes  = import ./runtimes.nix  { inherit lib pkgs; };
      utilities = import ./utilities.nix { inherit lib pkgs; };
      config    = builtins.fromTOML (builtins.readFile path);
      tools     = config.tools or {};
      env       = config.env   or {};

      # Resolution order for each tool:
      # 1. Check overrides (user-provided replacement) -- highest priority
      # 2. Check runtimes (version-specific resolution)
      # 3. Check utilities (direct pkgs.X mapping)
      # 4. Throw descriptive error (unknown tool)
      resolve = name: version:
        let v = builtins.toString version;
        in
          if overrides ? ${name} then overrides.${name}
          else if runtimes ? ${name} then runtimes.${name} v
          else if utilities ? ${name} then utilities.${name} v
          else builtins.throw
            "mise2nix: unknown tool '${name}' — not found in runtimes or utilities. Use 'overrides = { ${name} = pkgs.something; }' or 'extraPackages = [ pkgs.something ]' to provide it.";

      resolvedPackages = builtins.attrValues (builtins.mapAttrs resolve tools);
    in
      pkgs.mkShell {
        packages = resolvedPackages ++ extraPackages;
      };
}
```

Key changes from current lib/default.nix:
1. Function signature: `{ pkgs }:` becomes `{ pkgs, extraPackages ? [], overrides ? {} }:`
2. Import utilities.nix alongside runtimes.nix
3. Resolution uses a 4-step cascade: overrides -> runtimes -> utilities -> throw
4. No more null filtering -- every tool MUST resolve or throw (no silent drops)
5. `extraPackages` is appended to `packages` list after resolved tools
6. `overrides` values are derivations directly (NOT functions) -- the user passes `overrides = { mytool = pkgs.something; };` and we use the value as-is
7. Error message for unknown tools names the specific tool AND explains both overrides and extraPackages as escape hatches (per DX-01)

IMPORTANT: The existing flake.nix call `self.lib.fromMiseToml ./mise.toml { inherit pkgs; }` MUST continue to work without changes because extraPackages and overrides both default to empty. Verify this.

IMPORTANT: The overrides value is a derivation, not a function. `overrides = { node = pkgs.nodejs_20; }` replaces the resolved node package with nodejs_20 directly. This is simpler than making overrides a function and matches the "attrset keyed by mise tool name" requirement from CORE-04.
