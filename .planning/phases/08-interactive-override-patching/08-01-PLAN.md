# Phase 08 Plan 01: Interactive Override Patching — Wrapper Extension

## Objective

Extend the `miseWrapper` in `lib/default.nix` to detect unknown backends and unmapped
tools within known backends, prompt the user interactively for a nixpkgs attribute,
write the entry to `mise.toml`, and patch the `overrides = { ... }` argument in the
nearest `flake.nix`.

## Context

- `@lib/default.nix` — current miseWrapper let-binding (lines 82–129); extend this
- `@lib/backends/pipx.nix` — source of pipx tool names
- `@lib/backends/npm.nix` — source of npm tool names
- `@lib/backends/cargo.nix` — source of cargo tool names
- `@.planning/phases/08-interactive-override-patching/08-CONTEXT.md` — decisions

## Tasks

### Task 1: Add known-tool list derivations to lib/default.nix let block

Add `pipxKnown`, `npmKnown`, `cargoKnown` bindings derived from `builtins.attrNames`
on the existing backend attrsets (D-02). These get interpolated into the miseWrapper
bash script at Nix eval time — zero drift from the Nix tables.

**Done:** `lib/default.nix` let block contains `pipxKnown`, `npmKnown`, `cargoKnown`
as space-separated string lists derived from `builtins.concatStringsSep " " (builtins.attrNames ...)`.

### Task 2: Extend miseWrapper with unknown-tool detection and interactive prompt

Add detection logic after TOOL/VERSION extraction:
- If TOOL contains `:`, split into BACKEND and BARE_TOOL
- If BACKEND is not in `[pipx npm cargo]` → interactive path
- If BACKEND is in `[pipx npm cargo]` but BARE_TOOL not in the corresponding `*Known` list → interactive path
- Otherwise → existing mise.toml write path (known/mapped tool)

Interactive path:
1. Prompt: `Enter nixpkgs attribute for '<TOOL_SPEC>' (e.g. ripgrep or pkgs.ripgrep, Enter to cancel):`
2. Read input (trap SIGINT for Ctrl-C abort)
3. Empty input or Ctrl-C → print `[mise2nix] Cancelled.` and exit 0 (no files modified)
4. Strip leading `pkgs.` if present → `ATTR_NAME`
5. Proceed to mise.toml write (same as known path)
6. Also patch flake.nix overrides block

**Done:** miseWrapper bash script contains detection logic and interactive prompt;
existing known-tool path remains unchanged.

### Task 3: Implement flake.nix patching logic

After getting `ATTR_NAME` from interactive prompt, patch the nearest `flake.nix`:

Walk-up discovery (D-09): starting from `$PWD`, walk toward root to find first `flake.nix`.
- If none found: print warning and skip patching (files already written to mise.toml)

Patching strategy (D-07, D-08):
- If `overrides = {` exists in the file: use sed to append inside the block
- If no `overrides = {` exists: append a comment noting the user should add it manually
  (full injection of missing arg is fragile with sed; a clear hint is more robust)

Entry format: `"<TOOL_SPEC>" = pkgs.<ATTR_NAME>;`

**Done:** flake.nix patching runs after mise.toml write; handles missing flake.nix and
missing overrides block gracefully.
