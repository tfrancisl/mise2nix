# Phase 08 Plan 02: Interactive Override Patching — Check Derivations

## Objective

Add `nix flake check` derivations that verify the WRAP-03 interactive override patching
behavior: unknown-backend detection, unmapped-tool detection, abort on empty input, and
flake.nix patching via sed. All checks must pass without a TTY (Nix sandbox is
non-interactive).

## Context

- `@lib/default.nix` — miseWrapper with WRAP-03 interactive path (lines 119–232)
- `@flake.nix` — existing 22 checks; new checks go in the same `checks` output
- `@.planning/phases/08-interactive-override-patching/08-01-SUMMARY.md` — wrapper decisions

## Tasks

### Task 1: Add wrapper-unknown-backend-no-tty check

In a Nix sandbox there is no controlling terminal — `/dev/tty` is unavailable or
returns empty immediately. The `read -r NIX_ATTR </dev/tty` returns empty string → the
wrapper prints `[mise2nix] Cancelled.` and exits 0 with no file modifications.

Test: invoke the miseWrapper (inline duplicate, following RESEARCH.md pitfall 2
pattern established in phase 7) with `mise use "ubi:some-tool"`. Verify:
- Exit code is 0
- Output contains `[mise2nix] Cancelled.`
- No mise.toml created/modified

**Done:** `wrapper-unknown-backend-no-tty` check passes; output contains "Cancelled."
and no mise.toml was written.

### Task 2: Add wrapper-unmapped-known-backend-no-tty check

`pipx:nonexistent_xyz` is not in the pipxKnown list → NEEDS_PROMPT=1 → same abort
path as Task 1 (no TTY in sandbox).

Test: invoke miseWrapper with `mise use "pipx:nonexistent_xyz_abc"`. Verify:
- Output contains `[mise2nix] Cancelled.` (or is not in backend tables)
- No mise.toml written

**Done:** `wrapper-unmapped-known-backend-no-tty` check passes.

### Task 3: Add wrapper-known-tool-no-prompt check

`mise use "pipx:black"` is a known/mapped tool. The wrapper must NOT enter the
interactive path — it follows the WRAP-02 path: writes to mise.toml, prints
`[mise2nix]` message, no prompt.

Test: invoke miseWrapper with `MISE_CONFIG_FILE=mise.toml mise use "pipx:black"`.
Verify:
- mise.toml contains `"pipx:black" = "latest"`
- Output contains `[mise2nix]` attribution (not the "not in Nix backend tables" prompt message)

**Done:** `wrapper-known-tool-no-prompt` check passes; mise.toml written, no prompt triggered.

### Task 4: Add wrapper-flake-patch-overrides check

Verify the sed patching logic: given a flake.nix fixture containing `overrides = {`,
running the wrapper's sed command inserts a new override entry after that line.

Since interactive input is unavailable in sandbox, this check directly exercises the
sed patching step by:
1. Creating a flake.nix fixture with an `overrides = {` block
2. Running the same sed command used by the wrapper
3. Verifying the new entry appears in the patched flake.nix

**Done:** `wrapper-flake-patch-overrides` check passes; sed correctly inserts
`"ubi:some-tool" = pkgs.sometool;` after the `overrides = {` line.
