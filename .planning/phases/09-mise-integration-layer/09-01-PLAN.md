# Plan 09-01: lib/mise-installs.nix + MISE_INSTALLS_DIR

## Goal
Build a Nix derivation that mirrors the mise install directory structure for all resolved tools,
and wire it into lib/default.nix so `mise ls` shows tools as active.

## Changes

### NEW: lib/mise-installs.nix
Exports `mkMiseInstallsDir pkgs tools resolvedMap` — a `pkgs.runCommand` derivation with the
structure mise expects:
- Core tools (node, python, go, …): `{tool}/{ver}/bin/{binary}` — mise adds `bin/` to PATH
- All others (utilities, backends): `{tool}/{ver}/{binary}` — mise adds version dir to PATH

Binary symlinks are created via shell glob `ln -s "${pkg}/bin/"*` inside the build script,
avoiding eval-time IFD/readDir. Version dir named with the exact string from mise.toml.

### lib/default.nix
- Import mise-installs.nix
- Split `resolvedPackages` into `resolvedMap` (attrs) + `resolvedPackages` (values)
- Build `miseInstallsDir = miseInstalls.mkMiseInstallsDir pkgs tools resolvedMap`
- Replace `MISE_PREFER_OFFLINE = "true"` with `MISE_OFFLINE = "1"` (strict, no network fallback)
- Replace `MISE_SHARED_INSTALL_DIRS = []` with `MISE_INSTALLS_DIR = miseInstallsDir`
- Remove `MISE_AUTO_INSTALL`, `MISE_EXEC_AUTO_INSTALL` — keep `MISE_NOT_FOUND_AUTO_INSTALL`
